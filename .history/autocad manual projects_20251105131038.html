MEAM for AutoCAD — User Guide

How to install and use the MEAM command to manage drawing properties.

What is it?

MEAM is a manual AutoCAD command that opens a small form to view and update key drawing properties (DWG custom properties). It does not auto-run on file open.

Key points
  • Runs only when you type MEAM.
  • No popups at startup; no automatic property reads at load.
  • Works on any DWG; if a project number is detected, it’s shown to you (often read-only).

Install (one-time)

  1. Create the folder: C:\MEAM_AUTOCAD_SCRIPTS.
  2. Copy the provided MEAM.lsp into that folder.
  3. Add the folder to AutoCAD’s Support Path:
    • AutoCAD → Options → Files tab → Support File Search Path → Add → select C:\MEAM_AUTOCAD_SCRIPTS → OK.
  4. (Optional) Make the command available at every launch (still manual):
    • Type APPLOAD → Startup Suite → Contents… → Add… → pick MEAM.lsp → Close.

Use the command

  1. Open your DWG.
  2. Type MEAM in the command line and press Enter.
  3. The MEAM form opens. Review the fields, make changes, then click OK to save to DWG properties (or Cancel to exit without changes).
  4. Save the drawing as usual.

Fields in the MEAM form

The form writes to the DWG’s Custom properties (DWGPROPS → Custom). Here’s what each field means and how to update it:

Field What it means How to update
Project Number (may be read-only) The identifier of the project the drawing belongs to (e.g., P012345). Used to group documents and apply project conventions. If shown read-only, it was detected from the filename or set centrally; otherwise, edit as needed and follow your naming rules.
Document No (DocumentNo) Your unique document identifier for this DWG (e.g., numbering per internal procedure). Type or paste the correct document number. Keep formatting consistent with project rules.
Document Title (DocumentTitle) Human-readable title that describes the content or purpose of the drawing. Enter a clear, concise title (e.g., “General Arrangement – Dryer Section”). Avoid abbreviations that aren’t standard.
Approver (Approver) Name or initials of the person who approves the document for its intended status (working/review/issue). Enter the designated approver exactly as required by your QA procedure (initials vs full name).
Session Type (SessionType) The current working mode of this session. Typical options:
  • Working — regular editing
  • Updating — revising existing content
  • Reviewing — checking content (no major edits)
 Select the single option that best describes your session. This helps downstream tracking and status reports.
Requires New Review (RequiresNewReview) Flag indicating that recent changes require a fresh review cycle. Toggle Yes only when your edits invalidate a previous review or when requested by QA/lead.

Note: Some teams also track Phase, Scale, Rev, or ProjectName. If those appear, fill them per your local procedure. They are not required for basic use.

Troubleshooting

“Unknown command: MEAM”

  • Load the LISP: type APPLOAD and pick MEAM.lsp.
  • Check that C:\MEAM_AUTOCAD_SCRIPTS is in Support File Search Path.

Form doesn’t show / errors

  • Verify you loaded the correct MEAM.lsp and there are no command line errors.
  • Try another DWG to rule out a file issue.

Properties not saved

  • Check that the DWG is not read-only.
  • Click OK in the form, then save the drawing.

Project number looks wrong

  • It may be detected from the filename. Confirm your DWG naming.
  • If editable, correct it in the form per the project’s rules.

Support: Contact CAD Admin / your project lead for help with MEAM or property conventions.
